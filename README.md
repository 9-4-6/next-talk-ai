# next-talk-ai
是一款基于大语言模型（LLM）+ 工具调用（Tool）+ 向量数据库的下一代智能对话系统

## 项目简介
`next-talk-ai` 摒弃传统智能对话系统依赖「人工标注+意图分类模型」的老旧模式，采用「LLM + 工具调用 + 向量数据库」的全新架构，从根本上解决传统系统标注成本高、问法覆盖不全、扩展性差的核心痛点：
1. **核心逻辑**：将所有业务能力封装为标准化 Tool，通过向量数据库存储 Tool 的语义特征（核心描述+典型问法）；
2. **对话匹配**：用户提问时，通过向量语义检索直接匹配最贴合的 Tool（替代传统意图分类）；
3. **多轮交互**：基于会话上下文实现参数复用/重置，支持复杂业务场景的连续对话；
4. **持续优化**：自动采集用户真实问法，定期更新向量数据库，让系统「越用越准」。

## 核心特性
| 特性 | 详情 |
|------|------|
| 🚀 意图识别革新 | 向量语义检索替代传统机器学习意图分类，无需大量标注数据，仅需定义 Tool 核心描述 |
| 🔧 标准化 Tool 体系 | 支持多业务 Tool 封装（举例电力业务场景：如查电费/停电/报装/投诉/转人工），参数自动提取与复用 |
| 💬 多轮会话能力 | 内置会话上下文管理，支持参数复用、覆盖、重置，适配复杂业务交互 |
| 🔄 向量库智能更新 | 自动采集用户问法，定时增量更新向量库，动态适配用户口语化表述 |
| 🛡️ 双层匹配保障 | 向量检索 + 关键词兜底，兼顾语义泛化与匹配精准度 |

## 适用场景
- 电力/政务/金融等领域的智能客服系统重构；
- 需支持多业务工具调用的智能对话场景；
- 传统意图分类系统升级（解决标注成本高、扩展性差问题）。

## 技术栈
### 核心框架
- Spring Boot 3.x
- Spring AI 1.1.x

### LLM 适配
- 云端模型：deepseek
- 本地化模型：Llama 3

### 向量数据库
- Qdrant

### 核心组件
- Tool Search：工具语义检索核心
- 会话上下文管理：多轮参数复用/重置
- 向量增量更新：用户问法采集与向量库迭代
